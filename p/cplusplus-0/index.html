C++编译内存相关

从.cpp到.exe

.cpp源码 → 编译 → .s汇编代码 → 汇编 → .o机器指令 → 链接 → .exe可执行文件

.cpp文件里可能调用了另一个.cpp文件定义的函数或库函数，链接将调用的目标文件连接起来，生成.exe可执行文件。

静态链接v.s.动态链接

很多中文网页和教材讲的不好理解，其实主要区别就是静态在编译时链接，动态在运行时链接。

The main difference is that static linking copies all library modules used in the program into the final executable file at the final step of the compilation while, in dynamic linking, the linking occurs at run time when both executable files and libraries are placed in the memory.

静态链接 动态链接
编译时链接 运行时链接
.exe更大（打包了lib到exe） 节省内存
更新困难（一旦更新lib需重新链接） 更新方便
运行速度快 性能损失，每次执行程序都需链接

C++内存管理

内存分段

地址从低到高，代码段，bss段，数据段，堆区，栈区：

  • The code segment (also called a text segment)
    • where the compiled program sits in memory. The code segment is typically read-only.
  • The bss segment (also called the uninitialized data segment)
    • where zero-initialized global and static variables are stored.
  • The data segment (also called the initialized data segment)
    • where initialized global and static variables are stored.
  • The heap
    • where dynamically allocated variables are allocated from.
  • The call stack
    • where function parameters, local variables, and other function-related information are stored.

C++ 中不再区分初始化和未初始化的全局变量、静态变量的存储区。

#include <iostream>
using namespace std;

int g_var = 0; // g_var 在全局区（.data 段）
char *gp_var;  // gp_var 在全局区（.bss 段）

int main()
{
    int var;                    // var 在栈区
    char *p_var;                // p_var 在栈区
    char arr[] = "abc";         // arr 为数组变量，存储在栈区；"abc"为字符串常量，存储在常量区
    char *p_var1 = "123456";    // p_var1 在栈区；"123456"为字符串常量，存储在常量区
    static int s_var = 0;       // s_var 为静态变量，存在静态存储区（.data 段）
    p_var = (char *)malloc(10); // 分配得来的 10 个字节的区域在堆区
    free(p_var);
    return 0;
}

注意上面p_var在栈区，但指向的10个malloc分配的字节在堆区。

堆 / 栈

堆 栈
程序员管理 系统分配
空间不连续（类似链表） 空间连续（最大容量系统预定）
向高地址扩展 向低地址扩展

堆不连续的原因：系统自动查找内存中第一个合适的大小进行空间分配，会产生内存碎片。

static local

Using the static keyword on a local variable changes its duration from automatic duration to static duration. This means the variable is now created at the start of the program, and destroyed at the end of the program (just like a global variable). As a result, the static variable will retain its value even after it goes out of scope!

为什么这里会retain呢？因为加了static之后，变量是放在data区的，不加的时候，变量放在栈区，函数结束就被自动销毁。

#include <iostream>
void incrementAndPrint()
{
    int value{ 1 }; // automatic duration by default
    static int s_value{ 1 };// static duration via static keyword.  This initializer is only executed once.
    ++value;
    ++s_value;
    std::cout << value << ' ';
    std::cout << s_value << ' ';
} // value is destroyed here
// s_value is not destroyed here, but becomes inaccessible because it goes out of scope

int main()
{
    incrementAndPrint();
    incrementAndPrint();
    incrementAndPrint();
    return 0;
}

新建对象

建立方式

C++中对象建立方式有两种：

  • 静态建立：编译器在栈区分配内存，直接调用类的构造函数，如A a;，A是类名，a是对象名。
  • 动态建立：new先调用operator new()在堆上寻找合适的内存并分配，再调用类的构造函数，如A *p = new A()。

限制对象在堆或栈上

只能在堆上：将构造和析构函数设置为 protected，通过调用 create()函数在堆上创建对象。

内存对齐

默认对齐基数是8字节 #pragma pack(8)。

  • 结构体总大小需要为min(对齐基数，最大成员类型)的整数倍，否则在末尾加填充字节。
  • 结构体每个成员相对首地址的偏移量是min(对齐基数，该成员类型)的整数倍，否则在结构体中间加填充字节。 做内存对齐主要是为了适配硬件设备，某些硬件设备和处理器只支持对齐数据的存取。

类的大小

sizeof 计算规则

sizeof(A)返回类的实例化对象大小。

  • 遵循结构体对齐原则。
  • 与函数无关，与静态数据无关：因为他们被类的对象共享，并不属于哪个具体的对象。
  • 虚函数/虚继承对类的大小有影响。但虚函数的个数并不影响，因为对象只保存了指向虚函数表的指针。
  • 空类的大小为1。
  • 64位机器的指针占8字节（64/8bit/byte），32位机器的指针占4字节（32/8bit/byte）。

//64位编译器
#include <iostream>
using namespace std;

class A
{
private:
    static int s_var; // 不影响类的大小
    const int c_var;  // 4 字节
    int var;          // 8 字节 4 + 4 (int) = 8
    char var1;        // 12 字节 8 + 1 (char) + 3 (填充) = 12
public:
    A(int temp) : c_var(temp) {} // 不影响类的大小
    ~A() {}                      // 不影响类的大小
    virtual void f() { cout << "A::f" << endl; }
    virtual void h() { cout << "A::h" << endl; } // 24 字节 12 + 4 (填充) + 8 (指向虚函数的指针) = 24
};

int main()
{
    A ex1(4);
    A *p;
    cout << sizeof(p) << endl;   // 8 字节 注意：指针所占的空间和指针指向的数据类型无关
    cout << sizeof(ex1) << endl; // 24 字节
    return 0;
}
